# KIS-GALLERY — Инструкции для Claude

## Протокол Запуска Сессии (ОБЯЗАТЕЛЬНО)

При КАЖДОМ новом разговоре в этой папке, ВСЕГДА читай файлы в таком порядке:

### 1. Системная Логика
```
SYSTEM-LOGIC.md → Как работает система ABC-TOM
```

### 2. Ядро (C-core/) — КТО мы
```
C-core/project-brief.md → Цели и требования
C-core/tech-spec.md     → Техническая архитектура
```

### 3. Память (M-memory/) — ЧТО помним
```
M-memory/learning-log.md → Что узнали
M-memory/decisions.md    → Какие решения приняли
M-memory/feedback.md     → Какой фидбек получили
```

### 4. Агенты (A-agents/) — КАК работаем
```
A-agents/developer-agent.md → Для разработки
A-agents/reviewer-agent.md  → Для проверки
A-agents/content-agent.md   → Для контента
```

## Структура Проекта

```
KIS-GALLERY/
├── A-agents/        → Роли и инструкции агентов
├── B-brain/         → Знания, референсы, исследования
├── C-core/          → Ядро: цели, требования, спеки
├── T-tools/         → Инструменты и воркфлоу
├── O-output/        → Результаты работы (черновики)
├── M-memory/        → Память: уроки, решения, фидбек
├── src/             → Исходный код (готовый)
├── cms-admin/       → Код админки
└── public/          → Статика и конфиги
```

---

## Правила

1. **Всегда начинай с C-CORE** — пойми контекст
2. **Проверяй M-MEMORY** — не повторяй ошибки
3. **Используй правильного агента** — для каждой задачи свой
4. **Сохраняй в T-TOOLS** — если создал что-то полезное
5. **Обновляй M-MEMORY** — после каждой значимой работы
6. **O-OUTPUT → src/public** — сначала черновик, потом прод
