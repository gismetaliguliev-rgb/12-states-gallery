# Агенты KIS-GALLERY

Агенты — это роли, которые я "надеваю" для разных типов задач.

---

## Иерархия

```
ТЫ (Kis — направление + финальное одобрение)
    ↓
┌─────────────────────────────────────────────┐
│              АГЕНТЫ                         │
├─────────────────────────────────────────────┤
│  Developer ──► строит фичи, пишет код       │
│  Content   ──► добавляет фото, пишет текст  │
│  Reviewer  ──► проверяет качество           │
└─────────────────────────────────────────────┘
    ↓
ТЫ (одобряешь + деплоишь)
```

---

## Доступные Агенты

### Developer Agent
**Файл:** `developer-agent.md`
**Роль:** Пишет код, создаёт фичи, исправляет баги

**Активируется когда:**
- Нужно написать новый код
- Нужно исправить баг
- Нужно добавить функционал

**Читает:** C-core/*, M-memory/*, T-tools/components/*
**Создаёт:** O-output/*, src/*
**Обновляет:** M-memory/learning-log.md, M-memory/decisions.md

---

### Content Agent
**Файл:** `content-agent.md`
**Роль:** Управляет контентом галереи — фото, описания, расположение

**Активируется когда:**
- Нужно добавить фото
- Нужно написать описание
- Нужно изменить расположение фото

**Читает:** C-core/project-brief.md, public/gallery-config.json
**Создаёт/Изменяет:** public/gallery-config.json, public/assets/photos/*
**Обновляет:** M-memory/learning-log.md

---

### Reviewer Agent
**Файл:** `reviewer-agent.md`
**Роль:** Проверяет качество, производительность, соответствие стандартам

**Активируется когда:**
- Нужно проверить код
- Нужно оценить производительность
- Нужно найти проблемы

**Читает:** C-core/tech-spec.md, M-memory/*, O-output/* или src/*
**Создаёт:** O-output/[task]/review.md
**Обновляет:** M-memory/feedback.md, M-memory/learning-log.md

---

## Когда Какой Агент

| Задача | Агент |
|--------|-------|
| Написать новый код | Developer |
| Исправить баг | Developer |
| Добавить фичу | Developer |
| Добавить фото | Content |
| Написать описание | Content |
| Изменить расположение | Content |
| Проверить код | Reviewer |
| Оптимизировать | Reviewer |
| Найти проблемы | Reviewer |

---

## Как Агенты Работают Вместе

```
Пример: Добавить новую комнату в галерею

1. Developer создаёт код комнаты
   └──► сохраняет в O-output/new-room/

2. Reviewer проверяет код
   └──► создаёт O-output/new-room/review.md

3. Если одобрено:
   Developer переносит код в src/

4. Content добавляет фото в новую комнату
   └──► обновляет gallery-config.json
```
